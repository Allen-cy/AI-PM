# V6.6.9 个人飞书接入与正式试点准备度

## 目标

V6.6.7已经在执行端强制个人连接与组织事实源使用同一Base、同一领域表和同一同步流水表，但用户中心只能手工填写八个表ID，普通用户在保存和测试阶段看不到组织标准映射，容易到最终确认时才遇到范围冲突。

V6.6.9把该门禁前移到个人配置流程：

1. 根据当前有效业务角色确定组织；
2. 只向组织成员返回共享事实源的非敏感八表映射与配置来源；
3. 用户可一键复制项目、里程碑、任务、风险、合同、回款、成本、同步流水表ID；
4. App Secret和App Token永不从组织配置回显，用户仍需使用自己的飞书应用凭据并从有权限的同一Base取得App Token；
5. 连接测试精确比较Base Token和八张表，跨Base或表ID不同均失败关闭；
6. 写入权限测试继续要求浏览器二次确认，只向同步流水表写入诊断记录。

## 用户中心补齐

- 新增组织事实源对齐卡片和“复制组织八表映射”；
- 新增个人`open_id`或群聊`chat_id`通知接收对象配置，供风险提醒和个人通知复用；
- 飞书测试结果增加“组织事实源一致性”步骤；
- 当前用户无组织权限时返回403，不泄露其他组织表ID；
- 组织模板不包含App Secret、Base Token或任何密文；
- 375px下用户资料、AI模型、飞书凭据、八表映射、通知配置和操作按钮均为单列，无页面整体横向溢出。

## 数据库与业务边界

本版本不新增migration，继续使用现有`user_feishu_connections`加密字段、通知字段与`organization_feishu_connections`。组织共享配置仍负责读取和对账，消息、任务、文档及记录写回仍只使用当前用户个人连接，不回退管理员身份。

本版本只提升接入准备度，不创建真实个人连接，不执行237条分类，也不代替四位真实用户签字。正式试点仍需真实用户自行保存凭据、完成只读测试，并在明确确认后执行写入测试和业务确认。
