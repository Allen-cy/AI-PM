## 7. Supabase 判断

项目已经包含 Supabase 依赖、`src/lib/supabase.ts`、`src/lib/db.ts` 和 `supabase-schema.sql`。当前 V5.0.6 的业务事实源仍是飞书 Base，原因是本阶段目标集中在 PMO 协作、表单沉淀和飞书跳转。

建议后续数据架构：

- 飞书 Base：保留为业务协同层和人工录入层。
- Supabase：作为系统数据库，承载应用原生数据、权限、操作日志、RAG 检索状态、AI 任务队列和缓存。
- 双写/同步账本：通过现有同步账本记录幂等键、来源系统、处理状态和错误信息。

## 8. 业财一体化融合情况

已体现的位置：

- 飞书 Base 数据模型：项目、合同、回款、成本预算、风险等 8 张业务表。
- 项目组合看板：合同总额、回款总额、回款率、应催账款、账龄分组、成本健康矩阵。
- 合同与回款模块：合同金额、已回款、未回款、回款率、付款里程碑、未来回款预测。
- LTC 全流程：覆盖合同签约、合同管理、项目实施、结项、回款管理、运营运维，并包含财务/法务 RACI。
- 监控中心：EVM、CPI、成本偏差、预算消耗、变更成本影响。
- 知识库分类：已有 `业财一体化` 分类与相关问题。

不足：

- 目前多数页面仍是前端演示数据，尚未全部从飞书或 Supabase 动态读取。
- 业财指标尚未形成“合同-回款-成本-利润-项目健康度”的统一真实口径计算服务。
