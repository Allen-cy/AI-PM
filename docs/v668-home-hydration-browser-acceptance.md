# V6.6.8 登录后主页 hydration 与浏览器验收修复

## 问题

V6.6.7 生产浏览器验收从登录页开始监听页面错误。登录成功后应用会先进入主页，再打开飞书数据分类治理台；这段跳转捕获到 React #418 hydration mismatch。

主页此前直接在渲染阶段调用 `new Date().getHours()` 和 `new Date().toLocaleDateString()`。服务端输出与浏览器首次渲染可能位于不同时间、时区或日期边界，因此相同节点的文本不稳定。

## 修复

- 新增纯函数 `buildHomeClock(now)`，集中生成早上、下午、晚上的问候语和中文日期。
- 服务端与浏览器首次渲染统一使用 `INITIAL_HOME_CLOCK`，保证首屏文本完全一致。
- React 挂载后通过 `useEffect` 读取浏览器本地时间，再更新问候语和日期。
- 不使用 `suppressHydrationWarning` 掩盖不一致。

## 浏览器验收口径

- 从生产登录页开始，使用一次性 `test_account` 和默认组织 PMO 角色。
- 只打开主页和 `/integration-center/data-governance`，不点击“创建写回确认”或任何外部写入。
- 治理页必须等待自动目标对账说明出现，即真实快照已经加载，再统计按钮、筛选器和网络状态。
- 分别检查 1440×900 与 375×812；两者均不得出现页面横向溢出。
- 控制台错误、页面错误、失败请求必须为 0；治理 API 必须返回 200。
- 验收结束后删除临时会话、业务角色和账号。

## 数据库与业务边界

本版本不新增 migration，不改变 V6.6.7 的个人飞书身份门禁、同 Base 校验、单记录定向对账、幂等或失败恢复。现有隔离记录不被自动分类，真实飞书写回仍需个人连接和人工二次确认。
