# AI-PM System 任务日志

## 2026-07-15 | V6.5.0 跨角色、飞书、AI 与知识闭环

- **跨角色链：** PM/运营提交、PMO 复核与汇报冻结、CEO 决策、行动下发、执行回执、效果复核和关闭形成同一状态机。
- **事件一致性：** 领域状态与追加式 `business_events` 在同一事务写入；事件只能追加，不能更新或删除。
- **正式成果复用：** 报告工厂的正式汇报快照可直接进入跨角色决策链，不再重复录入。
- **飞书身份：** 组织连接只读共享 Base；消息、任务、文档和个人写回只使用操作人的个人连接，不回退到管理员身份。
- **角色 AI：** 支持定时扫描授权范围内的信号、风险和行动；建议保留证据、置信度、人工接受/驳回及效果评价。
- **动态知识：** 已发布知识条目和正式知识资产按组织、项目与数据分类进入 RAG；发布自动同步知识库状态并保护 215 组原始冗余。
- **数据库：** 四份 V6.5 migration 已应用生产；新增写接口同时要求幂等键与期望版本。
- **下一步：** V6.6.0 全模块验收候选、五项目隔离演练、两条黄金链与真实人员受控试点准备。

## 2026-07-15 | V6.4.0 四角色工作台与统一收件箱

- **角色分层：** PM、运营、PMO、CEO 使用共同数据底座，但首屏目标、事项排序和经营摘要按职责分离。
- **统一收件箱：** 聚合风险、联合检查、运营日历、治理审批、管理信号、AI 建议、决策回执、飞书确认和正式成果，并保存个人处理回执。
- **业务输入：** 项目、人员、证据、业务对象和正式成果统一使用业务名称选择器；复杂字段改为结构化编辑，不再要求用户输入 UUID 或 JSON。
- **飞书边界：** 风险通知只使用当前用户的个人飞书连接，缺少配置时明确引导，不静默借用管理员身份。
- **职责分离：** 建立四个不同测试账号、五个隔离测试项目以及 PM/运营→PMO→CEO 汇报关系；凭据只保存在本机忽略文件。
- **数据库：** `20260715220000_v640_role_workbench_inbox.sql` 已应用生产，四张新表 RLS 开启、客户端直接授权为 0、安全审计零违规。
- **验收边界：** 自动化账号只证明认证、权限和角色隔离，不冒充四个真实人员的试点签字。
- **下一步：** V6.5.0 跨角色业务事件、报告复用、组织/个人飞书、角色 AI 和动态 RAG 闭环。

## 2026-07-15 | V6.3.3 项目控制统一真实化

- **统一事实：** 执行、监控、风险、问题、变更、行动、质量和收尾读取同一授权项目快照。
- **真实数据：** 任务和里程碑来自飞书事实镜像；问题、变更、行动、质量和收尾来自 Supabase 正式状态，不再依赖硬编码页面数据。
- **人工闭环：** 风险升级、问题/变更状态变化、行动关闭由真实用户触发，责任人、期限、证据和版本贯穿事件链。
- **AI 边界：** 摘要与洞察由服务端重载真实快照；模型失败保留确定性事实，不伪造成功。
- **并发与审计：** 写入强制稳定项目 UUID、角色、数据分类、幂等键和期望版本，业务记录与追加式事件原子提交。
- **数据库：** 两份 V6.3.3 migration 已应用生产；完整安全审计零违规。
- **下一步：** V6.3.4 正式汇报、会议、迁移结果与知识成果持久化。
